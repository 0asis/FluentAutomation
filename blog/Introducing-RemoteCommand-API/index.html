Introducing the RemoteCommand API

I’ve mentioned the RemoteCommand API several times recently in posts, even on the Coming Soon portion of the site.

I think its time I explain what it is, why it exists and why I’m excited about it.

What

The RemoteCommand bits essentially expose a service endpoint that receives instructions in the form of a JSON document. The document contains a collection of Commands with the necessary Arguments to execute them.

Why

Any-Platform Test Creation

This allows for the origin of the tests to essentially be.. anything. Any language that can create well-formed JSON can run an automated UI test. We’re looking for contributors to help provide native fluent bindings for languages such as Ruby, PHP and Java.

An example of a Ruby test being built/executed via JSON is available here.

Playback from Fluent Automation Recorder

Another project that utilizes this API is the Fluent Automation Recorder by Paul Zumbrun. It allows you to instantly play back the test you’ve just recorded for verification.

Scalability

A common problem in UI Automation testing is the time involved in test runs. Anything that simulates user input is inherently slow compared to Unit Tests or even standard Integration Tests. As much as we try to speed things up, sometimes the only way to get through your tests in a reasonable manner is to scale out.

The native C# binding supports enabling Remote Execution which essentially targets a machine hosting the RemoteCommand service endpoint. This can be used for any number of scale out plans to spread the load of testing across several machines.

When

RemoteCommand support is in the master branch and in latest builds now, but not officially released. I’m working on the user experience of the FluentAutomation.Server (RemoteCommand endpoint) application to be useful in local debugging scenarios when tied with the Recorder.

I expect to have it out in the wild, ready to go in the next few weeks. The code is there for anyone who wants to try it out.

Let me know what you think of this upcoming feature!
